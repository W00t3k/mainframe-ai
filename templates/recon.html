▼ The Core Problem

Mainframes are assessed using incorrect OS assumptions. Failures are methodological, not technical. Exploits are rarely required when authority is misunderstood.

z/OS is not a server. It is a federation of subsystems where security decisions occur outside the kernel. Modern assessors import assumptions from Unix and cloud that do not apply. The result: critical exposure is missed not because testers lack skill, but because they ask the wrong questions.

▼ Broken Assumptions

5 assumptions modern assessors import that do not apply to z/OS:

Assumption
"There is a root user"

z/OS Reality
RACF distributes authority across profiles, not a single account. There is no omnipotent root -- authority is granted per-resource through profile definitions.
Assumption
"Processes are short-lived"

z/OS Reality
Address spaces persist. Identity outlives sessions. A CICS region may run for months. The security context you inherit depends on when and how you entered, not just who you are.
Assumption
"Ports define exposure"
Click to reveal z/OS reality
z/OS Reality
VTAM sessions outlive TCP connections. Network exposure does not equal authority. The session fabric exists independently of IP ports -- scanning nmap-style misses the real attack surface.
Assumption
"Authentication = Authorization"
Click to reveal z/OS reality
z/OS Reality
RACF separates these completely. Subsystems ask "may this identity do this action?" -- authentication proves who you are, but every resource access is a separate authorization decision.
Assumption
"Work executes immediately"

z/OS Reality
JES brokers deferred privileged execution. Jobs are declared, queued, and run later -- often under a different identity context. Authority decisions precede execution by minutes, hours, or days.

▼ The 5 Control Planes

z/OS security is not enforced by a single kernel. It is distributed across 5 control planes, each with its own identity model, enforcement boundary, and failure modes. Click a plane to jump to its assessment step.

TSO
TSO / ISPF
Human Interaction Plane
Interactive sessions, user identity, ISPF panels. Where humans enter the system and bind identity to a session.
JES
JES2 / JES3
Deferred Execution Plane
Job submission, scheduling, spool management. Work is declared before executed. Authority decisions precede execution.
RACF
RACF
Authorization Plane
Profiles, access control lists, identity management. The continuous authorization engine that every subsystem queries.
CICS
CICS
Transaction Execution Plane
Online transactions, regions, programs. Transactions are the real security boundary -- CICS mediates execution, not the OS.
VTAM
VTAM
Session Fabric Plane
LU sessions, APPLIDs, network layer. The session fabric that exists independently of TCP/IP. Sessions outlive connections.
