Overview

The Mainframe AI Assistant is a natural language interface for MVS mainframe operations. It runs entirely locally using Ollama, with no API keys required. Connect to real mainframes via TN3270, ask questions about JCL and ABEND codes, and get AI-powered assistance for security assessments.

Features

🎬 Autonomous Walkthrough

Watch live demonstrations of mainframe navigation with methodology narration.

🛡️ Recon & Assessment

Native TN3270 enumeration — TSO users, CICS transactions, VTAM applications.

🖥️ Terminal Console

Full TN3270 terminal with keyboard support, PF keys, and screen capture.

🔍 Mainframe Scanner

Network scanning for TN3270 services with banner grabbing and fingerprinting.

🤖 AI Chat Assistant

Ask questions about MVS, JCL, COBOL, ABEND codes with RAG-enhanced responses.

🕸️ Trust Graph

BloodHound-inspired visualization of jobs, datasets, and trust boundaries.

Quick Start

Setup (5 minutes)

# 1. Set up virtual environment
python3 -m venv venv
source venv/bin/activate

# 2. Install dependencies
pip install -r requirements.txt

# 3. Start Ollama (in another terminal)
ollama serve
ollama pull llama3.1:8b

# 4. Run the web app
python web_app.py

Then open http://127.0.0.1:8080 in your browser.

Demo Without a Mainframe

  • Start Ollama (ollama serve)
  • Run the web app (python web_app.py)
  • Open /chat, /tutor, and /graph
  • /terminal requires a TN3270 target (TK5 or a real mainframe)

TK5 Bundle (Local MVS)

The TK5 distribution provides a local MVS 3.8j environment for testing. Download from your MVS documentation and unzip into tk5/mvs-tk5. Connect to localhost:3270 after starting the emulator.

Modules

Walkthrough

Autonomous demonstrations that connect, login, and navigate through mainframe subsystems while narrating each screen through the control-plane methodology lens. Available walkthroughs:

  • Session Stack — VTAM → TSO → ISPF traversal, dataset browsing
  • Deferred Execution — JES queues, active jobs, output history
  • System Inspection — TSO sessions, system procedures, configuration
  • Authorization Model — RACF profiles, dataset protection
  • Dataset Model — PDS structure, catalogs, HLQ identity binding
  • Address Spaces — What's running, SDSF inspection
  • COBOL Development — Create datasets, edit source, compile workflow

Recon & Assessment

Native TN3270 enumeration capabilities:

  • TSO Enumeration — Discover valid userids via logon response analysis
  • CICS Enumeration — Probe transaction codes for valid responses
  • VTAM Enumeration — Discover APPLIDs and network resources
  • Hidden Fields — Detect non-display fields that may contain sensitive data
  • AI Analysis — LLM-powered screen analysis through methodology lens

Trust Graph

Visualize mainframe relationships in a BloodHound-inspired graph. Ingest JCL, SYSOUT, and screen captures to build a map of:

  • Jobs and their submitting identities
  • Programs and their execution contexts
  • Datasets and access permissions
  • Trust boundaries and privilege paths

Assessment Methodology

The assistant uses a control-plane assessment methodology designed for security professionals approaching mainframes from a Linux/Unix background. Key principles:

The 5 Assessment Questions

  1. Q1: Where is identity bound? — Identity binds at TSO logon, not VTAM entry
  2. Q2: When is authority evaluated? — At command execution, not session start
  3. Q3: What executes later than expected? — JCL submits now, runs later under submitter's identity
  4. Q4: Which subsystem enforces policy? — RACF, not the application or menu
  5. Q5: What assumptions are you importing? — Datasets ≠ files, address spaces ≠ processes

Control Planes

The mainframe operates across multiple control planes:

  • VTAM — Network entry, no identity yet
  • TSO — Interactive sessions, identity-bound
  • ISPF/RFE — Menu-driven interface over TSO
  • JES — Batch execution, deferred under submitter identity
  • RACF — Central security enforcement
  • CICS — Transaction processing subsystem
