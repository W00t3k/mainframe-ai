This Isn't New — The Model Predates the Tool

Temporal validation: the same mental model, different platforms

2018 — Active Directory & Enterprise Infra

"Attackers consistently succeed not because of exploits, but because defenders misunderstand where authority is actually enforced."

— Applied to AD delegation abuse, ADCS misconfigurations, service account sprawl

2024 — Mainframe / MVS

"Security is explicitly federated across subsystems. If you bring the wrong mental model, you miss the real attack paths."

— Same model applied to RACF, VTAM, JES, CICS trust boundaries

What Changed?

  • Not the model — same failure modes, same root causes
  • The platform — mainframes expose it more explicitly
  • The tooling — now operationalized in code

"The tool didn't invent the idea.
It operationalizes it."

Why This Matters

  • Proves the model generalizes across platforms
  • Not a one-off demo — years of validation
  • Same patterns: delegation, misplaced trust, authority confusion
Speaker Note: "In 2018, I was writing about how attackers succeed because defenders misunderstand where authority is enforced. At the time, it was AD. What changed isn't the model — what changed is that I now have a platform where the same failure modes are even more visible. Mainframes expose this clearly because security is explicitly federated across subsystems."
