Mental Models You'll Learn Foundational concepts

Mainframes reward people who build the right mental model: layered session state, control planes that enforce policy, and artifacts that persist long after commands run. This platform teaches those models through live TN3270 interaction, a step-gated tutor, and a trust graph that explains what each action touches (panels, jobs, programs, datasets). An LLM tutor backed by retrieval answers questions in context, using the current screen state and lab environment. The goal is not memorizing commands, but learning how to reason about MVS/mainframe systems the way experienced operators, auditors, and red teamers do.

  • Session Stack — VTAM → TSO → ISPF as layered state, not a single login.
  • Control Planes — RACF, JES, ISPF, CICS, DB2 determine access and execution.
  • Artifacts & Evidence — Spool, logs, datasets, and what survives execution.
  • Trust Boundaries — Dataset profiles, program paths, APF, and authorization.
  • Batch vs Interactive — Why JCL behaves differently than panels and commands.
  • Graph Thinking — Panels, jobs, programs, datasets as nodes with edges.

Behavioral Expectations

  • Selecting a module or tutor informs the LLM of the mental model focus.
  • Steps are gated by terminal state, not by free-form conversation.
  • Questions always reference the current screen and lab environment.

Mental Model Walkthrough

  • Start at a TN3270 login screen.
  • Show panel → transaction → job → dataset relationships.
  • Walk a user to a concrete goal (e.g., DB2 access, job submit, credential abuse).
  • Explain why each step exists historically.
  • Map each action to a mental model (not MITRE).
